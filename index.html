🔑 GitHub API Token Guide

For certain advanced features (e.g., managing blog posts via an admin interface), you might need a GitHub Personal Access Token (PAT). Here's how to get one:

  1. Log in to your GitHub account.
  2. Click your profile picture in the top right corner and select "Settings".
  3. In the left sidebar, choose "Developer settings".
  4. Click on "Personal access tokens" and then "Tokens (classic)".
  5. Click "Generate new token" and select "Generate new token (classic)".
  6. Give your token a descriptive note (e.g., "You Are Blogging Admin").
  7. Select the necessary scopes (permissions). For repository content access, you'll need at least `repo` (for private repositories) or `public_repo` (for public repositories). For content management, `workflow` is often useful.
  8. Click "Generate token".
  9. Copy the generated token IMMEDIATELY. You won't be able to see it again later!

For more detailed information, refer to the official GitHub documentation:
